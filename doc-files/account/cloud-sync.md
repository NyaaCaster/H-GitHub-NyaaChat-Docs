# 云同步

云同步让你在多台设备间接续使用 NyaaChat——在一台设备上备份设置和聊天记录，在另一台设备上一键恢复。

## 云同步能同步什么

云同步支持三类数据的独立上传和下载：

| 同步内容 | 包含什么 | 是否加密 | 传输方式 |
|----------|----------|:------:|----------|
| **设置** | 供应商配置（含 API Key）、外观偏好、角色列表元数据、Bypass 配置、用户人设、发送模式等 | 🔒 **Nyaa-HMAC-XOR-V1 加密** | `PUT/GET` 云端 |
| **角色封面** | 每个角色的封面图片（WebP 格式） | 明文 | `PUT/GET` 云端 |
| **聊天记录** | 完整的聊天会话列表 | 🔒 **Nyaa-HMAC-XOR-V1 加密** | `PUT/GET` 云端 |


## 前提条件

**必须已登录 NyaaAcount**。未登录时点击云同步按钮会自动弹出登录窗口。

同一 NyaaAcount 账号在不同设备上共享同一套云同步数据——上传和下载都基于你当前的登录账号。

## 入口

云同步有两个入口：

1. **设置 → 备份与恢复**：在设置弹窗（⚙️ 齿轮图标）中，找到"备份与恢复"区域。这里有四个按钮：
   - ☁️ **上传设置** — 将本地设置和封面备份到云端
   - ☁️ **下载设置** — 从云端恢复设置和封面
   - 💾 **导出设置** — 下载为本地 JSON 文件（不经过云端）
   - 📥 **导入设置** — 从本地 JSON 文件恢复（不经过云端）

2. **聊天记录弹窗**：在聊天记录弹窗（📋 聊天记录图标）底部，有三个按钮：
   - ☁️ **上传记录** — 加密后上传聊天记录到云端
   - ☁️ **下载记录** — 从云端下载并解密聊天记录
   - 📥 **导入聊天记录** — 从本地文件导入聊天记录

## 备份到云端（上传）

### 上传设置

1. 打开设置弹窗 → 备份与恢复 → 点击 **"上传设置"**。
2. 系统会先检查云端是否已有存档。如果已有，会弹出确认对话框，显示上次上传的时间。
3. 确认后，本地设置（JSON）和所有角色封面（WebP）会自动上传。
4. 封面上传是 best-effort：如果某个封面上传失败，不影响设置上传的成功。

### 上传聊天记录

1. 打开聊天记录弹窗 → 点击底部 **"上传记录"**。
2. 确认弹窗 → 系统自动用 **AES-256-GCM** 加密所有聊天会话。
3. 加密后的数据上传到云端。

> 🔑 加密密钥由 NyaaChat 服务端自动生成，与你的 NyaaAcount 账号绑定。同一账号在不同设备上使用**同一密钥**——所以你在设备 A 上传的记录，设备 B 可以用同一密钥解密下载。

## 从云端恢复（下载）

### 下载设置

1. 打开设置弹窗 → 备份与恢复 → 点击 **"下载设置"**。
2. 系统会检查云端存档状态，弹出确认对话框。
3. 确认后，**本地所有设置会被云端存档覆盖**——包括供应商配置、角色列表元数据、Bypass 配置、用户人设、发送模式等。
4. 随后自动下载所有角色封面。

> ⚠️ **重要**：下载设置会**完全替换**你当前的本地设置。如果本地有未上传的新配置，请先上传到云端再在另一设备下载。

### 下载聊天记录

1. 打开聊天记录弹窗 → 点击底部 **"下载记录"**。
2. 确认弹窗 → 从云端下载加密数据 → 自动用 AES-256-GCM 解密。
3. 解密后**替换本地所有聊天会话**。

> ⚠️ **重要**：下载聊天记录会**完全替换**你当前的本地聊天记录。如果本地有未上传的新聊天，请先上传。

## 安全注意事项

| 方面 | 说明 |
|------|------|
| 聊天记录加密 | 聊天记录使用 **🔒 **Nyaa-HMAC-XOR-V1 加密**** 端到端加密。密钥账户级共享，只有你的 NyaaAcount 账号能解密。 |
| 导出/导入 | **纯本地操作**，数据不经过任何服务器。导出的 JSON 文件含 API Key 明文——请妥善保管，不要分享给他人。 |

导出文件命名格式为 `NyaaChatSetting-YYMMDDhhmmss.json`，建议存到安全位置。

## 导出导入（本地备份）

除了云端同步，NyaaChat 还支持纯本地的导出/导入，无需网络、不需要登录：

- **导出设置**：下载一个包含全部设置的 JSON 文件到本地。在你需要备份、重装系统、或迁移数据时很有用。
- **导入设置**：选择一个之前导出的 JSON 文件，**覆盖当前所有本地设置**。
- **导出/导入聊天记录**：同理，在聊天记录弹窗中可以将聊天记录导出为文件或从文件导入。

> ⚠️ 导入操作会**覆盖**当前数据，请确认后再操作。

## 多端使用注意事项

1. **单向流动**：先在设备 A 上传，再到设备 B 下载。不要在两端同时上传——后上传的会**覆盖**先上传的存档。
2. **加密密钥**：服务端自动管理，同一 NyaaAcount 账号下所有设备共享同一密钥。你不需要手动管理密钥。
3. **封面是 best-effort**：如果下载设置后某个角色的封面没恢复，重新上传一次封面即可。
4. **聊天记录建议先传后清**：切换设备前，先在旧设备上传聊天记录，再到新设备下载。

## 常见问题

**上传失败了？**  
检查是否已登录。如果登录过期（token 失效），重新登录后再试。

**下载设置后角色封面丢失？**  
封面是 best-effort（不影响设置上传成功）。回到原设备重新上传一次封面，再到新设备下载。

**下载设置后本地配置全没了？**  
下载设置会**完全覆盖**本地配置。这是预期行为。如果想保留本地配置，建议先在本地**导出设置**（纯本地备份），再执行下载。
