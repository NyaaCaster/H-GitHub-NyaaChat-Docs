# NyaaChat 是什么

NyaaChat 是一个**在浏览器中运行的本地 AI 角色扮演聊天应用**。打开浏览器即可使用，无需安装客户端或部署服务器。

它不自带大语言模型——你需要搭配一个 API 供应商（比如 QinyAPI、OpenAI、Claude 等），用自己的 API Key 来驱动对话。所有数据保存在你的浏览器本地，不会被传到官方服务器。

## 一句话定位

如果你知道 [SillyTavern](https://github.com/SillyTavern/SillyTavern)，可以把 NyaaChat 理解为**它的精神继承者**：继承了 SillyTavern 的角色卡格式与提示词架构，但使用 React + TypeScript 全栈重写，拥有独立的账号体系、知识库系统和更现代的界面交互。

如果你没用过 SillyTavern，可以把它理解为一个**自由度极高的 AI 陪聊工具**——你可以创建任何你能想象的角色，为它编写世界规则与背景设定，上传文档让它基于你的资料应答，并与朋友共享你精心调试的角色卡。

## 它能帮你做什么

NyaaChat 围绕四个核心板块设计：

- **开始使用** — 从零到第一次对话，配好模型，熟悉界面。这也是你当前正在读的内容。
- **角色扮演** — 创建、编辑、导入、分享角色。为角色配上世界书规则、正则脚本、自定义变量，打造深度沉浸的对话体验。
- **知识库** — 上传文档构建 RAG 知识库，让 AI 基于你的专属资料来回答。绑定到特定角色后，检索结果会自动注入到对话上下文中。
- **账号系统** — 登录 NyaaAcount 后可以使用猫粮充值、共享角色库、云同步等需要联网的功能。不登录也可以离线使用全部本地方案。

> **扩展**功能（兼容 SillyTavern 扩展）暂未对外开放，本文档不做介绍。

## 隐私与数据模型

NyaaChat 的数据模型非常简单：

- **你的数据存在你的浏览器里**。角色、聊天记录、设置、API Key 等全部保存在浏览器本地的 IndexedDB 数据库中，不会上传到 NyaaChat 官方服务器。
- **对话内容直接发送到你配置的 API 供应商**（如 QinyAPI、OpenAI 等），由该供应商处理并返回回复。NyaaChat 不经手中转你的对话内容。
- **账号功能是可选补充**。登录 NyaaAcount 后才会有跨设备同步、共享角色库等云端能力，但这些不是必须的——离线单机使用同样完整可用。

## 自带 API 与供应商

NyaaChat 预置了以下对话模型供应商的配置模板：

| 供应商 | 说明 |
|--------|------|
| **QinyAPI** | 国内推荐，性价比高，无需特殊网络环境，同时提供对话模型和生图模型 |
| OpenAI | 需要 API Key，可能需要特殊网络 |
| Anthropic Claude | 需要 API Key，支持代码生成、图像理解等 |
| Google Gemini | 需要 API Key，提供免费额度 |
| DeepSeek | 需要 API Key，性价比高 |
| Ollama | 本地运行，无需 API Key，适合自建模型 |

你只需要在一个供应商处获取 API Key，填入 NyaaChat 即可开始使用。推荐新用户从 QinyAPI 开始——配置最简单，国内访问最稳定。

## 与 SillyTavern 的关系

如果你正在从 SillyTavern 迁移：

- **NyaaChat 支持导入 SillyTavern 格式的角色卡**（PNG 角色卡和 JSON 格式），你的现有角色可以直接迁移过来。
- **世界书（World Info）** 的迁移：NyaaChat 的角色规则功能与 ST 世界书高度兼容，迁移后按 ST 原有逻辑运行。
- **提示词架构继承自 ST**，包括宏变量、系统提示词注入位置等都保持兼容。
- **不同的是**：NyaaChat 是单页 Web 应用（不需要 Node.js 环境），拥有内置的账号系统、知识库、云同步等 ST 不具备的功能，UI 交互基于现代弹窗模式而非模块页面。
