# 配置对话模型

NyaaChat 不自带大语言模型，使用前必须配置至少一个对话模型供应商。本文以 **QinyAPI** 为主线，带你完成从零到可用的全部配置步骤。

## 为什么需要先配置模型

简单说：NyaaChat 是一个"对话界面"，它负责把你说的话组织成 AI 能理解的格式、发给 AI、再把回复展示给你。但"理解并回复"这件事本身是由大语言模型（LLM）完成的——而这些模型运行在供应商的服务器上。

你需要：
1. 在一个 API 供应商处注册账号并获取 **API Key**（密钥）。
2. 在 NyaaChat 中填入这个 Key，告诉它"请用这个供应商的模型来对话"。

QinyAPI 是目前推荐的首选——**注册简单、国内网络直连、价格实惠**，且同时提供对话模型和生图模型。

## 打开模型配置弹窗

1. 点击顶部工具栏第一行右侧的 **设置** 按钮（⚙️ 齿轮图标），打开设置弹窗。
2. 在设置弹窗中找到 **对话模型供应商** 一栏。

> 注：设置弹窗中还有"生图模型供应商"和"嵌入模型供应商"，本文只涉及对话模型。生图配置请参见对应板块；嵌入模型配置请参见 [知识库：前置准备](../knowledge-base/prerequisites)。

## 以 QinyAPI 为例

以下是完整的分步指南。如果你使用其他供应商（OpenAI、Claude 等），流程类似，只是具体字段略有不同。

### 第一步：选择 QinyAPI

在设置弹窗左侧的供应商列表中，点击 **QinyAPI**。

> 💡 **关于拼写**：在 NyaaChat 界面中，QinyAPI 的接入点标签显示为 "QingAPI"。这只是一个历史拼写差异——二者是同一个供应商，本文统一使用 QinyAPI 这个名称。

### 第二步：选择接入点

QinyAPI 提供两个接入点：

| 接入点 | 地址 | 适用场景 |
|--------|------|----------|
| `.icu` | `love.qinyan.icu` | 国内网络推荐，延迟更低 |
| `.com` | `openai.chatnewai.com` | 海外部署，需特殊网络环境 |

点击 **QingAPI 接入点** 旁边的按钮，选择 `.icu`（国内推荐）。

### 第三步：填写 API Key

1. 登录 QinyAPI 平台，在个人中心或 API 管理页面获取你的 API Key。
2. 将 Key 粘贴到 **API Key** 输入框中。

API Key 默认以掩码形式显示（`sk-...`）。如果你需要确认粘贴是否正确，可以**按住**输入框右侧的眼睛图标来查看明文——松开后自动恢复掩码。

> ⚠️ API Key 相当于你的账户密码，请勿分享给他人。NyaaChat 将其存储在你的浏览器本地，不会上传到任何第三方服务器。

### 第四步：启用模型

1. 点击 **管理模型** 按钮，弹出一个新的管理弹窗。
2. NyaaChat 会自动从 QinyAPI 拉取当前可用的模型列表。
3. 勾选你想启用的模型。推荐初次使用时勾选以下几个：
   - `gemini-2.5-pro` — Google 旗舰模型，综合能力强
   - `gpt-4o` — OpenAI 多模态模型，支持图片理解
4. 关闭管理弹窗，选中的模型即会自动出现在输入区的模型选择卡片中。

> 模型列表会随时间更新，你可以随时打开"管理模型"查看新增的模型并勾选启用。

### 第五步：测试连接

关闭设置弹窗前，可以点击 **健康测试** 按钮来验证配置是否成功。测试通过后，你就可以开始对话了。

如果测试失败，请检查：
- API Key 是否正确粘贴（无多余空格）
- 接入点选择是否正确
- QinyAPI 账户是否余额充足

## 其他供应商

除了 QinyAPI，NyaaChat 还预置了以下几个供应商的配置模板。操作流程与 QinyAPI 类似——选择供应商 → 填 API 地址和 Key → 管理模型 → 测试。

| 供应商 | 关键注意事项 |
|--------|-------------|
| **OpenAI** | 使用 OpenAI 官方的 API 地址。需要特殊网络环境。API Key 格式为 `sk-...` |
| **Anthropic Claude** | 使用 Anthropic 官方地址。支持代码生成、图像理解。需要特殊网络环境 |
| **Google Gemini** | 使用 Google AI Studio 获取 API Key（免费额度）。国内可能需要特殊网络 |
| **DeepSeek** | 国内友好，价格实惠。API 兼容 OpenAI 格式，选择 OpenAI 兼容模式 |
| **Ollama** | 本地运行，无需 API Key。地址默认为 `http://localhost:11434`。需自行下载模型 |

自定义供应商（选择"API 兼容模式"后填写自定义地址）也可接入其他任何兼容 OpenAI API 格式的服务。

## 常见配置问题

### API Key 无效 / 返回 401 错误

最常见的原因：
- Key 粘贴时多带了空格或换行
- Key 已过期或被吊销（在 QinyAPI 平台重新生成）
- 接入点选错（尝试切换到另一个接入点）

### 模型列表拉取失败 / 超时

可能原因：
- 接入点不可达（尝试切换 `.com` ↔ `.icu`）
- 网络代理/防火墙问题
- API Key 配置不正确

### 接入点选哪个好

如果你在国内，选择 `.icu`（国内专用节点，延迟更低）。如果在海外或使用代理网络，`.com` 可能会更快。两个接入点提供的模型列表和价格相同。

### 用哪个模型好

这取决于你的需求：
- **日常对话和角色扮演**：`gemini-2.5-pro` 或 `gpt-4o` 是稳健的选择
- **逻辑推理和代码**：Claude 系列或 `gpt-4o`
- **预算有限**：DeepSeek 或 QinyAPI 上较便宜的模型
- **需要图片理解**：`gpt-4o` 或 Gemini 2.5 Pro（均支持多模态）

建议同时启用 2-3 个模型，根据场景在输入区快速切换。
