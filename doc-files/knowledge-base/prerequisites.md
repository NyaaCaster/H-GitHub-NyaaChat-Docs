# 前置准备

使用知识库功能之前，你需要完成两项准备：**登录 NyaaAcount 账号**，以及**配置一个嵌入模型**。前者解锁知识库功能的入口，后者让系统具备"理解文档语义"的能力。

## 需要先登录账号

知识库功能是**登录门控**的——你需要先登录 NyaaAcount 才能使用。

打开知识库管理弹窗（顶部工具栏 📖 按钮）时，如果尚未登录，你会看到一个登录入口：

- 提示文字：`请先登录以使用知识库功能`
- 按钮：**"去登录"**（点击后打开用户账号弹窗）

登录后，知识库管理弹窗会自动显示你的知识库列表。

> **注意**：对话、角色管理等功能不要求登录也能使用。只有知识库、共享角色库等功能需要登录。

登录和注册的完整操作见 [登录与注册](../account/login-register)。

## 什么是嵌入模型

嵌入模型（Embedding Model）是一个将**文本转换成数字向量**的 AI 模型。它本身不生成对话——它的唯一工作是把一段文字映射到高维空间中的一个"坐标"。

你可以这样理解：

> 把"苹果"和"水果"这两段文字交给嵌入模型，它们会得到两个很接近的向量坐标。把"苹果"和"汽车"交给它，得到的坐标则会很远。语义越相近 → 向量距离越近 → 检索时就能被找到。

NyaaChat 的默认嵌入模型是 `Qwen/Qwen3-Embedding-8B`，通过 SiliconFlow 提供。你当然也可以使用其他兼容的嵌入服务。

> 💡 **提示**：嵌入模型和对话模型是**两个不同的东西**。对话模型负责生成文字回复（如你配置的 QinyAPI），嵌入模型负责把文字变成可搜索的向量。你需要分别为它们配置 API Key——嵌入的 Key 独立于对话/图片的 Key。

## 配置嵌入模型

嵌入模型在「嵌入模型配置」弹窗中设置。以下按步骤说明。

### 打开嵌入配置弹窗

有两种方式打开嵌入配置：

- **方式一（手动）**：在知识库管理弹窗的标题栏右侧，点击 ⚙️ 齿轮图标。
- **方式二（自动）**：如果你首次打开知识库管理弹窗且尚未配置嵌入模型，配置弹窗会在短暂延迟后自动弹出。

弹窗标题为「嵌入模型配置」，包含三个配置字段。

### 填写嵌入供应商信息

| 字段 | 说明 | 占位符 / 默认值 |
|------|------|----------------|
| **API Base URL** | 嵌入 API 的端点地址 | 占位符：`https://api.siliconflow.cn/v1` |
| **API Key** | 你的 API 密钥 | 占位符：`输入 API Key` |
| **模型名称** | 嵌入模型的标识符 | 默认：`Qwen/Qwen3-Embedding-8B` |

几点说明：

- API Base URL 填嵌入服务提供的基础地址，需兼容 OpenAI 的 `/v1/embeddings` 端点格式。
- API Key 旁有一个 **"获取嵌入模型"** 链接按钮（外部链接图标），点击会跳转到 SiliconFlow 的注册/获取页面。
- 如果之前已保存过 API Key，则该字段会显示为 `已设置（留空保持不变）`，并出现一个 **"修改"** 按钮供你更换密钥。
- 模型名称需填写你在嵌入服务商处开通的模型 ID。不同供应商的模型名称格式不同，请按供应商的实际名称填入。

### 健康检查与保存

填写完以上信息后，需要先通过健康检查才能保存配置：

1. 点击左侧的 **"健康检查"** 按钮。
2. 系统会用你填写的配置向嵌入 API 发送测试请求。按钮状态变化：

| 状态 | 按钮表现 | 说明 |
|------|----------|------|
| 空闲 | 正常 **"健康检查"** 按钮 | 尚未检查或上次结果已过期 |
| 检查中 | 禁用 + 旋转加载图标 | 等待 API 响应 |
| 通过 | 绿色对勾 + 提示 `健康检查通过，维度: N` | 配置可用 |
| 失败 | 红色叉号 + 错误信息 | 配置有误，需排查 |

3. 健康检查通过后，**"保存设置"** 按钮变为可用。点击保存即可。

> ⚠️ 如果你**更换了模型名称**（比如从 `Qwen3-Embedding-8B` 换成其他模型），页面上会显示一条琥珀色警告：`更换嵌入模型将使已有知识库的向量失效，需要重新处理所有文档。` 这意味着你之前上传到知识库的文档需要重新上传才能被检索到。

> **注意**：底部有一个灰色提示 `请先通过健康检查，方可保存设置`——在健康检查通过之前，保存按钮始终禁用。关闭弹窗时如果有未保存的更改，系统会自动保存（即使没有通过健康检查），但建议手动通过检查后再关闭。

## 嵌入供应商与对话/图片供应商的关系

**嵌入模型供应商是完全独立的**。它和你配置的对话模型供应商、图片生成供应商互不影响。

举个例子，下面这种组合完全正常工作：

- 🤖 对话模型 → QinyAPI（API Key A）
- 🖼️ 图片生成 → Google（API Key B）
- 🔍 嵌入模型 → SiliconFlow（API Key C）

三个服务彼此独立，你不需要"统一用一家"。任何兼容 OpenAI `/v1/embeddings` 接口的服务都可以用作嵌入供应商——不限于 SiliconFlow。

> 💡 **提示**：嵌入 API Key 由你自行获取。NyaaChat 不提供嵌入服务，也不会替你申请 Key。

## 常见前置问题

**一定要用 SiliconFlow 吗？**  
不。任何兼容 OpenAI `/v1/embeddings` 接口的服务都可以。只需在「API Base URL」中填入对应服务的地址即可。SiliconFlow 只是默认推荐。

**为什么健康检查失败？**  
最常见的三种原因：① API Base URL 填错（多了一个 `/`、少了 `/v1`、用了 HTTP 而不是 HTTPS）；② API Key 不正确或已过期；③ 模型名称填错（你在供应商处开通的模型 ID 与你填写的不一致）。请逐一排查。

**"服务器无法连接，请检查 API Base URL"怎么办？**  
这个提示说明网络层面无法到达你填写的地址。检查 URL 是否正确、网络是否正常、供应商服务是否可用。

**更换嵌入模型后，已有知识库会怎样？**  
所有已有知识库的向量数据将失效。你需要重新上传每个知识库中的文档——系统会用新模型重新为它们生成向量。文档本身不会丢失，只是向量需要重建。

**API Key 安全吗？会被看到吗？**  
API Key 仅保存在知识库后端服务器上。前端界面只显示"已设置"的状态，不会回传完整的 Key。角色导出文件中也不会包含你的 API Key。

**嵌入模型需要自己付费吗？**  
取决于你使用的嵌入服务商。SiliconFlow 等供应商有自己的定价策略（通常按处理的 token 数计费）。NyaaChat 不收取嵌入费用——你支付的猫粮仅用于扩容知识库栈。
