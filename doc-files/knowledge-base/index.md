# 什么是知识库

知识库是 NyaaChat 的 RAG（检索增强生成）功能——它让你上传自己的文档，在对话中自动检索相关内容并注入给 AI。简单说：**给 AI 配一本可以随时查阅的参考书**。

## 一句话说清知识库

想象你在和 AI 跑团。你写了一本 200 页的《魔法学院设定集》——学院历史、课程体系、魔法派系、教授名单……但每次对话只涉及其中一小部分。如果把整本设定集塞进上下文，很快就会被截断。

知识库的做法是：**把设定集拆成小块，对话时只检索与当前话题相关的那几块，临时注入**。不相关的内容不占位置，相关的自动出现。

这就是 RAG 的核心思想——**不是把所有资料都喂给 AI，而是在需要时才去查**。

## RAG 是怎么工作的（通俗解释）

从文档到对话中的检索结果，整个过程分为五个步骤：

1. **上传文档**：你把自己的文本文件（`.txt`、`.md`、`.pdf`）上传到知识库。
2. **切分成分块**：系统把文档按语义段落切成小块（默认每块约 512 个字符，相邻块之间有 50 个字符的重叠，避免在句子中间截断）。
3. **嵌入为向量**：每块文本通过嵌入模型转换为一串数字（向量）。语义相近的文本 → 向量在空间中距离更近。
4. **建立索引**：所有向量存入向量数据库，同时建立全文索引（BM25），等待查询。
5. **对话时检索**：当你发送消息时，如果你的消息命中了关联知识库的角色规则，系统会把你的消息当作查询 → 在向量数据库中搜索最相关的分块 → 将结果注入到对话上下文中 → AI 依据这些参考内容生成回复。

> 💡 **提示**：以上过程全部自动完成。你只需要上传文档并绑定角色，剩下的——分块、嵌入、检索、注入——都由系统在后台处理。

检索到的分块会被包装成一个 `<search_context>` 块，追加到你的消息末尾。AI 会被告知这些内容是参考资料，可以引用但不必全盘接受。详细的注入格式见 [绑定角色与检索注入](./link-to-character)。

## 知识库与世界书的区别

知识库和世界书（World Info）都是"按需注入信息到对话"的机制，但它们的定位不同：

| 维度 | 知识库 | 世界书（World Info） |
|------|--------|---------------------|
| 内容来源 | 你上传的文档（`.txt` / `.md` / `.pdf`） | 你手动编写的规则文本 |
| 激活方式 | 对话内容自动**语义检索**——检索与当前话题最相关的分块 | **关键词匹配**或**永久激活** |
| 注入位置 | 用户消息末尾 `<search_context>` 块（作为参考资料） | 系统提示词顶部或助手消息前（作为指令/设定） |
| 适合场景 | 参考材料：设定文档、角色面板、百科条目、背景资料 | 行为规则：场景描述、角色约束、即时指令 |
| 内容大小 | 单个知识库可达数 MB 的文档 | 每条规则几百到几千字 |

**知识库是世界书的"资料库"**——你可以在世界书规则中关联知识库，当规则触发时，知识库同时被检索。两者配合使用，能让 AI 既「知道大量背景信息」又「遵守特定的行为规范」。

## 使用知识库的整体流程

1. **登录 NyaaAcount**——知识库功能需要登录才能使用。
2. **配置嵌入模型**——设置 API Base URL、API Key 和模型名称（详见 [前置准备](./prerequisites)）。
3. **创建知识库**——在知识库管理弹窗中点击「新建知识库」（详见 [创建与管理知识库](./manage)）。
4. **上传文档**——将你的 `.txt`、`.md` 或 `.pdf` 文件上传到知识库（详见 [上传与管理文档](./documents)）。
5. **绑定角色规则**——在角色编辑器的规则条目中，将知识库关联到触发关键词（详见 [绑定角色与检索注入](./link-to-character)）。
6. **对话**——当对话命中了关联知识库的规则时，系统自动检索并注入相关分块。你不需要做任何额外操作。

> **注意**：知识库的创建数量受额度限制，初始免费 3 个，超出后需要扩容。详见 [知识库栈额度与扩容](./quota)。
