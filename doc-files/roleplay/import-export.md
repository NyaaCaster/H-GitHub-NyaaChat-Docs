# 角色导入与导出

NyaaChat 支持导入和导出角色卡，让你可以在设备间迁移角色，或与其他玩家交换角色。本文介绍支持的格式、操作步骤和兼容性注意事项。

## 支持的格式

NyaaChat 支持两种角色卡格式，全部以 **PNG 图片文件**为载体——角色封面作为图片像素，角色数据（JSON）编码在 PNG 文件内部的 tEXt 数据块（chunk）中。也就是说，你看到的角色卡"图片"本身就是一个完整的角色数据包。

### NyaaChat 原生格式

**推荐使用**的格式，完整保留 NyaaChat 的全部高级功能：

- 导出文件命名：`NyaaChatChar-{角色名}-YYMMDDhhmmss.png`
- 完整保留的信息：角色名、描述、开场白、世界书规则、约束强度（软设定/硬约束）、绑定的正则脚本、角色变量、扩展数据
- 封面图片编码为 PNG 像素，数据 JSON 存储在 tEXt `chara` chunk 中

> ✅ **建议**：在 NyaaChat 用户之间互传角色时，始终使用原生格式——不会丢失任何数据。

### SillyTavern 格式

兼容 SillyTavern（ST）生态的角色卡格式：

- 导出文件命名：`SillyTavernChar-{角色名}-YYMMDDhhmmss.png`
- 格式标识：`"spec": "chara_card_v3"`
- 可导入的信息：角色名、描述、第一条消息、世界书条目
- **会丢失的信息**：
  - ⚠️ 约束强度（软设定/硬约束）→ 降级为默认行为
  - ⚠️ 绑定的正则脚本 → 不导出
  - ⚠️ 角色变量 → 不导出

> 从 SillyTavern 迁移角色到 NyaaChat 时使用此格式。导入后建议检查并重新配置丢失的高级规则。

## 导入角色

### 操作步骤

1. 打开 **角色选择** 弹窗（顶栏第二行角色名按钮）。
2. 点击底部左侧的 📥 **"导入角色"** 按钮。
3. 在文件选择器中选择一个 `.png` 角色卡文件。
4. NyaaChat 自动检测格式 → 解析 → 加入本地角色列表。

### 导入限制与校验

| 检查项 | 限制 |
|--------|------|
| 文件类型 | 仅 `.png` |
| 文件大小 | ≤ **5 MB** |
| 格式检测 | 自动判别 NyaaChat 原生 vs SillyTavern 格式 |
| 存储空间 | 预估导入后用量超过配额 95% 时阻止导入 |

### 导入错误提示

| 错误 | 提示文案 |
|------|----------|
| 文件过大 | `文件过大（X.XX MB），上限 5 MB` |
| 非 PNG 文件 | `不是有效的 PNG 文件` |
| 无角色数据 | `PNG 文件中未找到角色数据（chara chunk）` |
| 文件损坏 | `PNG 文件已损坏或被篡改` |
| 格式不对 | `仅支持 PNG 角色卡` |
| 储存不足 | 弹出"储存空间不足"对话框 → 前往扩容 |
| 其他错误 | `角色卡导入失败：` + 具体原因 |

### 导入后的存储检查

导入前系统会预估导入后的存储占用（`当前用量 + 文件大小 × 2`，因为 JSON 和封面 WebP 各存一份）。如果超过当前配额的 95%（默认 32 MB 上限），导入会被阻止并弹出扩容提示。

> 扩容角色卡储存：5 猫粮 / +12 MB。详见 [账号系统：额度与扩容](../account/quota-expand)。

## 导出角色

### 操作步骤

1. 打开 **角色选择** 弹窗 → 悬停目标角色卡片 → 点击 ✏️ 编辑图标。
2. 在角色编辑弹窗底部左侧，点击 📥 **"角色导出"** 按钮。
3. 弹出一个格式选择弹窗，标题为 **"选择导出格式"**。
4. 选择格式，浏览器自动下载 PNG 文件。

### 两种导出格式的对比

| | NyaaChat 格式 | SillyTavern 格式 |
|------|:--:|:--:|
| 文件名 | `NyaaChatChar-角色名-时间戳.png` | `SillyTavernChar-角色名-时间戳.png` |
| 约束强度 | ✅ 保留 | ❌ 丢失 |
| 正则绑定 | ✅ 保留 | ❌ 丢失 |
| 角色变量 | ✅ 保留 | ❌ 丢失 |
| ST 兼容 | ❌ 仅 NyaaChat 可读 | ✅ ST 可读 |
| **推荐场景** | NyaaChat 用户间互传 | 从 NyaaChat 导出给 ST 用户 |

## 兼容性与注意事项

### SillyTavern → NyaaChat（导入）

- 角色名、描述、开场白、世界书：✅ 完整导入。
- 高级规则（约束强度、正则、变量）：ST 不支持，导入后需在 NyaaChat 中手动补配。

### NyaaChat → SillyTavern（导出）

- 基础字段和世界书：✅ 兼容导出。
- 约束强度：⚠️ 丢弃——ST 世界书没有硬约束概念。导出后在 ST 中所有规则均为普通世界书条目。
- 正则脚本、角色变量：❌ 不兼容——需在 ST 侧用扩展或脚本重新实现。

### 建议

- **日常备份**：导出为 NyaaChat 原生格式，存档一份 PNG 文件。
- **分享给 NyaaChat 用户**：使用 NyaaChat 原生格式。
- **分享给 ST 用户**：使用 SillyTavern 格式（但不保证高级功能兼容）。
- **从 ST 迁移**：直接导入 PNG 角色卡，导入后手动补配高级规则。
