# 角色变量

角色变量是在对话过程中**动态变化的数值**——比如好感度、生命值（HP）、任务进度、金币数量等等。它们让角色不只是会说固定的话，而是能感知到"状态的变化"。

## 什么是角色变量

变量本质上是**一组键值对**——一个名字（键）对应一个数值（值）。例如：

```
affection: 50
hp: 100
mp: 30
quest_progress: 2
```

这些值在角色卡中定义初始值，在对话过程中通过正则脚本来更新。AI 角色可以读取这些值并在回复中体现——比如当 `affection` 从 50 涨到 80 时，角色对你的语气会肉眼可见地从冷淡变亲昵。

角色变量是 NyaaChat 特有的功能，在 SillyTavern 中不直接支持。

## 变量的定义与初始化

角色变量定义在角色卡的 JSON 数据中（`variables` 字段）。目前没有直接的 UI 界面来创建变量——你需要通过角色卡的 JSON 来设置，或在创建角色时预设。

如果在正则规则中引用了一个不存在的变量，它会被自动创建，初始值为 0。

## 在对话中读取与更新变量

### 读取变量

在角色文本中（角色描述、开场白、世界书内容、正则替换文本），使用 `{{变量名}}` 语法引用变量的当前值。例如：

- 角色开场白中写：`"你对我的好感度是 {{affection}} 分。"` → AI 开场时实际输出 `"你对我的好感度是 50 分。"`
- 世界书规则内容中写：`当前任务进度：{{quest_progress}}/10` → 触发时动态注入当前进度

这种方法让你可以让角色的文本**随状态自动变化**，而不需要手动修改角色卡。

### 更新变量

变量值通过**正则规则中的脚本**来更新。在正则替换文本中，可以使用 `setVariable()` 函数来修改变量的值：

- `setVariable("affection", +5)` — 好感度增加 5
- `setVariable("hp", -10)` — 生命值减少 10
- `setVariable("gold", 100)` — 金币设为 100（覆盖）

更新发生在正则规则的 placement 指定的阶段——比如在"发送给 AI 前"（placement 2）更新变量，AI 收到消息时就能看到最新的变量值。

### 典型用法：好感度系统

1. 角色编辑器中定义变量初始值：`affection: 50`
2. 创建正则规则 A（全局作用域、placement 2、"发送给 AI 前"）：
   - 查找：`谢谢|感谢|多谢`
   - 替换逻辑：检测到这些词时调用 `setVariable("affection", +1)`，文本不变
3. 角色文本中引用：`{{affection}}`
4. 效果：每次你说"谢谢"，好感度 +1。角色的回复中会体现好感度的变化。

## 与宏、世界书的配合

| 配合方式 | 说明 |
|----------|------|
| **变量 → 世界书** | 世界书规则中引用 `{{affection}}`，根据变量值展示不同的规则内容 |
| **变量 → 正则** | 正则中同时做文本替换和变量更新 |
| **宏** | `{{char}}`、`{{user}}`、`{{persona}}` 是**系统级"宏"**（不可修改，值固定），是变量的特例 |
| **多层联动** | 变量 `affection` > 50 → 正则触发特殊对话脚本 → 世界书注入更亲昵的规则 |

一个典型的联动链：用户高好感度 → 世界书注入高好感度规则 → 正则自动在 AI 回复中添加温暖描述 → 角色整体语气变化。

## 查看与重置变量

**查看当前值**：目前没有内置的 UI 界面来查看所有角色变量的当前值。最直接的方法是在输入框输入 `/status`（如果你设置了对应的斜杠命令），或者导出角色卡 JSON 来检查。

**重置变量**：
- 重新导入角色卡：重新导入同一角色的 PNG 角色卡，会恢复到卡中定义的初始值（但会丢失当前对话进度）。
- 删除并重新创建角色：相当于重新开始，所有变量回初始值。

> 💡 如果你想让变量在对话中持久化但又能查看，建议在角色卡的第一条消息中列出所有变量和初始值，同时在正则中设置一个 `/status` 斜杠命令来实时查看。
