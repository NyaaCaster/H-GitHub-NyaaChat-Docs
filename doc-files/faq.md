# 常见问题

这里汇总了 NyaaChat 使用过程中最常见的问题。每个问题下方有简短答案，并附带指向详细文档的链接。

## 账号与登录

**为什么登录后还是看不到某些功能？**  
登录不会自动提升你的各项额度上限。如果你需要更多共享卡槽、知识库栈或储存空间，需要花费猫粮扩容。详见 [额度与扩容](./account/quota-expand)。

**可以用同一个账号在多台设备登录吗？**  
可以。在任意设备上用同一个 NyaaAcount 账号登录即可。配合 [云同步](./account/cloud-sync) 功能，可以在不同设备间同步角色设定和聊天记录。

**Token 过期了怎么办？**  
登录状态会自动清除。重新打开用户账号面板（🪪 按钮），输入凭据重新登录即可。Token 过期不会影响你本地的角色和聊天数据。

**登录时提示"用户名或密码错误"但确定没输错？**  
确认你是在 NyaaAcount 注册的账号（不是在 NyaaChat 内注册——NyaaChat 没有独立的注册功能）。如果忘记了密码，前往 [NyaaAcount](http://h.nyaa.host:5110)使用找回密码功能。

## 猫粮与额度

**猫粮充了没到账？**  
先重新打开账号面板刷新数据。如果仍未到账，退出登录再重新登录。NyaaAcount 的支付回调可能有短暂延迟，稍等片刻后再查看。

**猫粮能转给别人吗？会过期吗？**  
猫粮与 NyaaAcount 账号绑定，不支持转账。猫粮永久有效，不会过期。

**扩容扣了猫粮但上限没变？**  
退出账号面板再重新打开，或退出登录后重新登录——上限数字会从 NyaaAcount 重新查询。如果仍然不一致，请联系 NyaaAcount 支持。

**哪些功能需要扩容？**  
五项功能设有初始免费上限：共享卡槽（初始免费数个）、知识库栈（初始 3 个）、角色卡储存（32 MB）、聊天记录储存（32 MB）、NyaaComfyUI 图包（10 次生图）。达到上限后需要花费猫粮扩容。详见 [额度与扩容](./account/quota-expand)。

**扩容是永久的吗？**  
是的。扩容增加的上限永久保留，不会随时间减少或被回收。

## 对话模型配置

**API Key 提示无效或返回 401 错误？**  
最常见的原因：① Key 复制时带入了多余空格；② Key 已过期或被注销；③ 接入点（Base URL）填错。逐一排查这三项。

**模型列表拉取失败或超时？**  
通常是因为接入点不可达——检查 Base URL 是否正确、网络是否正常、是否存在防火墙阻断。如果你在国内使用 `.com` 接入点，尝试换为 `.icu` 接入点。

**该用哪个模型？**  
没有"最好"的模型，取决于你的需求：角色扮演推荐 gemini-2.5-pro 或 claude-opus-4-8（创意和情感表现力强）；需要严谨逻辑推理时推荐 Claude 或 GPT-4o；预算敏感时 DeepSeek 性价比高。详见 [配置对话模型](./guide/configure-model)。

**对话模型和嵌入模型是同一个 API Key 吗？**  
不是。对话模型负责生成回复，嵌入模型负责知识库的语义搜索——它们是完全独立的两个服务，需要分别配置各自的 API Key。详见 [前置准备](./knowledge-base/prerequisites)。

## 角色扮演

**角色导入后和 SillyTavern 中表现不一样？**  
NyaaChat 兼容 SillyTavern 的角色卡格式（chara_card_v2/v3），但少数特性不互通：NyaaChat 的硬约束规则在导出为 ST 格式时会丢失；ST 的一些专属字段（如 Lorebook 的 scan depth）在导入 NyaaChat 时会用默认行为替代。详见 [角色导入与导出](./roleplay/import-export)。

**世界书关键词不触发？**  
检查三点：① 关键词是否拼写正确（触发时大小写不敏感）；② 规则条目的开关是否开启（关闭的规则不参与匹配）；③ 你的消息中是否确实包含了这些关键词。使用通配符 `*` 表示始终匹配（等同于永久触发）。

**角色卡储存空间满了怎么办？**  
在账号面板中扩容角色卡储存（5 猫粮 / +12 MB），或删除不再需要的角色卡来释放空间。详见 [额度与扩容](./account/quota-expand)。

## 知识库

**为什么 AI 有时会忽略知识库检索到的内容？**  
AI 有权选择忽略不相关的结果。如果检索到的分块与当前问题匹配度不高，AI 可能不引用它们。尝试调整提问方式使其与文档内容更紧密相关。你也可以在控制台（`>_` 按钮）查看检索日志确认检索是否成功。

**知识库和世界书有什么区别？我应该用哪个？**  
知识库存放**参考材料**（你上传的文档），通过语义检索自动找到相关内容。世界书存放**行为规则**（你手写的指令），通过关键词匹配触发。两者互补——你可以把知识库关联到世界书规则上，让规则触发时同时检索知识库。详见 [什么是知识库](./knowledge-base/)。

**上传的 PDF 检索不到内容？**  
PDF 可能只包含扫描图片而没有文本层。只有包含文本层的 PDF（如电子书、导出的文档）才能被正确提取。扫描版 PDF 需要先用 OCR 工具转换。

**知识库最多能创建几个？**  
初始免费 3 个，每次扩容 +2 个（花费 10 猫粮），硬上限 50 个。详见 [知识库栈额度与扩容](./knowledge-base/quota)。

**使用共享角色时，关联的知识库检索会消耗我的额度吗？**  
不会。共享角色的跨账号知识库检索完全独立于你的个人知识库栈额度，不需要额外配置。详见 [绑定角色与检索注入](./knowledge-base/link-to-character)。

**更换嵌入模型后，已有知识库会怎样？**  
所有已有知识库的向量数据将失效——新模型生成的向量与旧模型不兼容。你需要重新上传每个知识库中的文档，系统会用新模型重新生成向量。文档本身不会丢失。

## 云同步与数据

**上传设置后下载，本地配置全没了？**  
这是预期行为。从云端下载设置会**覆盖**本地当前配置。如果你希望保留本地状态的同时尝试云端设置，建议先在本地做一次导出备份。详见 [云同步](./account/cloud-sync)。

**下载设置后角色封面丢失？**  
角色封面图片是尽力交付的（best-effort）——某些情况下原始图片无法通过云端恢复。从原设备重新上传封面即可。

**导出的 JSON 文件包含敏感信息吗？**  
是的。导出的设置文件可能包含明文 API Key 等敏感信息。请将导出的文件存放在安全位置（如加密目录或密码管理器），不要通过不安全的渠道分享。

**聊天记录和角色数据存在哪里？**  
NyaaChat 使用浏览器本地存储（IndexedDB）。你的聊天记录、角色卡、设置等数据都在你自己的浏览器中，不上传到任何中心服务器。这也意味着清除浏览器数据会导致数据丢失——建议定期通过云同步或导出功能备份。

## 其他

**NyaaChat 收费吗？**  
NyaaChat 本身完全免费开源。你需要付费的部分：① 对话/嵌入/生图模型的 API 费用（付给模型供应商，如 QinyAPI、SiliconFlow 等）；② NyaaAcount 的猫粮充值（仅用于扩容功能上限，不强制）。如果你只用本地模型（Ollama），可以不花一分钱。

**NyaaChat 和 SillyTavern 是什么关系？**  
NyaaChat 是 SillyTavern 的精神继承者，用 React 完全重写了前端，去除了对 Node.js 服务端的依赖，是一个纯浏览器端单页应用。它兼容 SillyTavern 的角色卡格式（chara_card_v2/v3），但并非 ST 的直接分支。

**数据会泄露给第三方吗？**  
NyaaChat 本身是纯前端应用——你的对话消息直接从浏览器发送到你配置的模型 API 供应商，不经过 NyaaChat 的服务器。你的 API Key 和聊天数据存储在浏览器本地。NyaaAcount 登录仅用于额度/同步等附加功能，不收集对话内容。
